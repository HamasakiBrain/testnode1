# Журнал работ — Строительный объект

Простое fullstack-приложение для учёта выполненных работ на строительном объекте. Позволяет прорабу фиксировать: дату, вид работ, объём и исполнителя.

## Стек

- **Next.js 16** (App Router) — полнофункциональный React-фреймворк. Выбран, чтобы в одном проекте объединить фронтенд и бэкенд (API Routes), не настраивая отдельный сервер.
- **React 19 + TypeScript** — типобезопасный UI.
- **Tailwind CSS + HeroUI** — современная доступная библиотека компонентов с хорошей типизацией и встроенной адаптивностью.
- **Prisma ORM + SQLite** — типобезопасная работа с базой данных. SQLite выбрана для максимально простого развертывания: не требует Docker или отдельного сервера БД.
- **TanStack Query** — кэширование, фоновые обновления и оптимистичные обновления при работе с API.
- **React Hook Form + Zod** — производительные формы с декларативной валидацией.

## Запуск

```bash
# 1. Установить зависимости
npm install

# 2. Применить миграции БД и заполнить справочник видов работ
npx prisma migrate dev --name init
npm run seed

# 3. Собрать и запустить production-сервер
npm run build
npm start
```

Приложение будет доступно по адресу http://localhost:3000.

Для разработки можно использовать `npm run dev` вместо `npm run build && npm start`.

## Функционал

- **Список записей** — таблица с датой, видом работ, объёмом и исполнителем.
- **Фильтрация по дате** — диапазон «с» и «по».
- **Сортировка** — по дате (сначала новые / сначала старые).
- **CRUD** — создание, редактирование, удаление записей.
- **Справочник видов работ** — выбор из предзаполненного списка (8 видов работ), хранится в отдельной таблице БД.
- **Валидация** — все поля формы обязательны, объём должен быть положительным числом.

## Структура проекта

```
src/
  app/
    api/
      log-entries/      — API для записей журнала (GET, POST)
      log-entries/[id]/ — API для отдельной записи (GET, PUT, DELETE)
      work-types/       — API справочника видов работ
    page.tsx            — главная страница
    layout.tsx          — корневой layout с провайдерами
    providers.tsx       — QueryClient провайдер
  components/
    LogEntriesTable.tsx — таблица записей
    LogEntryForm.tsx    — форма создания/редактирования
    DeleteConfirmModal.tsx — подтверждение удаления
  hooks/
    useLogEntries.ts    — хуки TanStack Query для записей
    useWorkTypes.ts     — хук TanStack Query для справочника
  lib/
    prisma.ts           — singleton PrismaClient
  types/
    index.ts            — TypeScript-типы
prisma/
  schema.prisma         — схема базы данных
  seed.ts               — seed справочника видов работ
```
